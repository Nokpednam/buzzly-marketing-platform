# Deep Dive: Audit Logs (/dev/audit-logs)

เอกสารสรุปฟังก์ชันการทำงานฝั่ง Backend และ Logic ของหน้า Audit Logs สำหรับนำไปใช้ประกอบการทำสไลด์

---

## 📊 1. สรุปกิจกรรมระบบ (Activity Stats)
ส่วนที่แสดงสถิติกิจกรรมสำคัญเป็นตัวเลขด้านบนสุดของหน้าจอ

| ชื่อฟังก์ชัน | Path ไฟล์ | รายละเอียดการทำงานและวัตถุประสงค์ |
| :--- | :--- | :--- |
| `useAuditLogStats` | `src/hooks/useAuditLogs.tsx` | **การทำงาน:** ดึงสถิติจำนวนกิจกรรมแยกตามหมวดหมู่สำคัญ (เช่น Login, Export, Security) จาก View `audit_logs_enhanced`<br>**วัตถุประสงค์:** เพื่อให้ผู้ดูแลระบบเห็น "ปริมาณการใช้งาน" และ "ความผิดปกติ" (เช่น Failed Login) ได้ในทันที |

---

## 📋 2. รายการบันทึกกิจกรรม (Activity Logs Table)
ส่วนแสดงประวัติการทำงานอย่างละเอียดของผู้ใช้ทุกคนในระบบ

| ชื่อฟังก์ชัน | Path ไฟล์ | รายละเอียดการทำงานและวัตถุประสงค์ |
| :--- | :--- | :--- |
| `useAuditLogs` | `src/hooks/useAuditLogs.tsx` | **การทำงาน:** ดึงข้อมูลจาก View `audit_logs_view` แบบแบ่งหน้า (Pagination) และ **"เลือกกรองข้อมูลได้หลายเงื่อนไขพร้อมกัน"**<br>**วัตถุประสงค์:** เพื่อใช้ในการ "ตรวจสอบย้อนหลัง" (Auditing) โดยระบุเงื่อนไขได้ทั้ง หมวดหมู่, บทบาท, สถานะ หรือค้นหาจากชื่อคน |

---

## 🛠️ 3. ตรรกะการกรองข้อมูล (Filtering Logic)
ความสามารถในการเจาะจงข้อมูลที่ต้องการตรวจสอบ

| หัวข้อ Logic | รายละเอียดการทำงาน | วัตถุประสงค์ |
| :--- | :--- | :--- |
| **Category Mapping** | แปลงหมวดหมู่จาก UI (เช่น Data) เป็นกลุ่ม Category ใน Database (เช่น report, export, import) | เพื่อให้การค้นหาครอบคลุมกิจกรรมที่เกี่ยวข้องทั้งหมดในคลิกเดียว |
| **Role & Action Filter** | กรองข้อมูลตามบทบาท (Owner, Dev, Support) และรูปแบบการกระทำ (Page View, Login) | เพื่อใช้จำกัดวงการตรวจสอบเมื่อเกิดเหตุการณ์ต้องสงสัยในกลุ่มผู้ใช้บางประเภท |

---

## 🗄️ 4. โครงสร้างข้อมูลเบื้องหลัง (Backend View)
เทคนิคที่ใช้ในการเพิ่มประสิทธิภาพการดึงข้อมูล

| ชื่อเทคโนโลยี | รายละเอียด | วัตถุประสงค์ |
| :--- | :--- | :--- |
| **Database View** | ใช้ `audit_logs_view` และ `audit_logs_enhanced` แทนการดึงจากตารางตรงๆ | เพื่อรวบรวมข้อมูลจากหลายตาราง (Logs + Users + Roles) ให้พร้อมใช้งานทันที ลดภาระของระบบและเพิ่มความเร็วในการโหลดหน้าจอ |

---

> [!TIP]
> **Performance Optimization:** มีการใช้ตัวเลือก `head: true` ในการนับจำนวนสถิติ (`useAuditLogStats`) ซึ่งจะช่วยให้นับจำนวนข้อมูลมหาศาลได้โดยไม่ต้องดึงตัวข้อมูลจริงๆ ออกมา ช่วยลดปริมาณ Data Transfer และทำให้ Dashboard โหลดเร็วขึ้นมาก
