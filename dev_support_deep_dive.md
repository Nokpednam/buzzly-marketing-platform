# Deep Dive: Support & Error Logs (/dev/support)

เอกสารสรุปฟังก์ชันการทำงานฝั่ง Backend และ Logic ของหน้า Support & Error Logs สำหรับนำไปใช้ประกอบการทำสไลด์

---

## 📋 1. รายการ Log และการกรองข้อมูล (Log Table & Filtering)
ส่วนที่แสดงรายการความผิดพลาดทั้งหมดในระบบ พร้อมรายละเอียดผู้ใช้งาน

| ชื่อฟังก์ชัน | Path ไฟล์ | รายละเอียดการทำงานและวัตถุประสงค์ |
| :--- | :--- | :--- |
| `useDevErrorLogs` | `src/hooks/useDevSupport.tsx` | **การทำงาน:** ดึงรายการ Log แบบแบ่งหน้า (Pagination) และมีระบบ **"แยกแยะรูปแบบการค้นหาอัตโนมัติ"**<br>**วัตถุประสงค์:** เพื่อให้นักพัฒนาค้นหาผลลัพธ์ได้ทันที โดยไม่ต้องเลือกหมวดหมู่ (พิมพ์รหัส ID ก็หาจาก ID, พิมพ์ข้อความปกติก็หาจากเนื้อหา Error ให้เอง) |
| `fetchUserMap` | `src/hooks/useDevSupport.tsx` | **การทำงาน:** เป็นระบบดึงข้อมูล "อีเมล" และ "บทบาท" จากตาราง `employees` และ `customer`<br>**วัตถุประสงค์:** เพื่อเปลี่ยนรหัส ID ที่อ่านยากให้เป็น "ข้อมูลผู้ใช้งานจริง" ทำให้รู้ว่า Error กระทบใครบ้าง |

---

## 📈 2. สรุปสถิติความรุนแรง (Stats Overview)
ส่วนที่แสดง Card จำนวน Error แยกตามระดับความสำคัญ

| ชื่อฟังก์ชัน | Path ไฟล์ | รายละเอียดการทำงานและวัตถุประสงค์ |
| :--- | :--- | :--- |
| `useDevLogStats` | `src/hooks/useDevSupport.tsx` | **การทำงาน:** คำนวณจำนวน Log ทั้งหมดในระบบแยกตาม Level (Critical, Error, Warning, Info)<br>**วัตถุประสงค์:** เพื่อประเมิน "ระดับสุขภาพของระบบ" ในภาพรวมว่ามีปัญหาที่ต้องแก้ไขเร่งด่วน (Critical) อยู่จำนวนเท่าใด |

---

## 🧪 3. การจำลอง Error (Error Simulation)
เครื่องมือสำหรับทดสอบระบบ Logging และ Notification

| ชื่อฟังก์ชัน | Path ไฟล์ | รายละเอียดการทำงานและวัตถุประสงค์ |
| :--- | :--- | :--- |
| `handleSimulateError` | `src/pages/dev/DevSupport.tsx` | **การทำงาน:** สร้าง Error จำลองขึ้นมาในระบบเมื่อกดปุ่ม "Test Error"<br>**วัตถุประสงค์:** เพื่อตรวจสอบว่า "ระบบ Alert" ยังทำงานปกติหรือไม่ และเพื่อให้นักพัฒนาเห็นตัวอย่างการพ่นข้อความ Error เข้าสู่ฐานข้อมูล |
| `logError` (Service) | `src/services/errorLogger.ts` | **การทำงาน:** ฟังก์ชันกลางที่ใช้รับข้อมูล Error แล้วนำไป Insert ลงตาราง `error_logs`<br>**วัตถุประสงค์:** เป็นจุดศูนย์รวมของ Logic การเก็บ Error ทั้งหมดในโปรไฟล์ (เก็บทั้ง message, stack trace และ metadata อื่นๆ) |

---

> [!TIP]
> **Performance Optimization:** หน้าขอนี้มีการใช้เทคนิค `keepPreviousData` ใน React Query เพื่อให้การสลับหน้า (Pagination) ลื่นไหล ไม่กระตุก และข้อมูลจะไม่หายไปจากหน้าจอในขณะที่กำลังโหลดข้อมูลหน้าใหม่
