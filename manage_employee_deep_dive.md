# Deep Dive: Manage Employee (/dev/employees)

เอกสารสรุปฟังก์ชันการทำงานฝั่ง Backend และ Logic ของหน้า Manage Employee สำหรับนำไปใช้ประกอบการทำสไลด์

---

## 👥 1. การจัดการพนักงาน (Employee CRUD)
ระบบจัดการข้อมูลพื้นฐานและการเพิ่ม/ลดบุคลากรในระบบ

| ชื่อฟังก์ชัน / Mutation | Path ไฟล์ | รายละเอียดการทำงานและวัตถุประสงค์ |
| :--- | :--- | :--- |
| `useEmployees` (Fetch) | `src/hooks/useEmployees.tsx` | **การทำงาน:** ดึงรายชื่อพนักงานทั้งหมด พร้อมข้อมูลโปรไฟล์และบทบาท (Role) แบบ Joins<br>**วัตถุประสงค์:** เพื่อแสดงภาพรวมพนักงานทั้งหมดในระบบพร้อมสถานะปัจจุบัน |
| `createEmployee` | `src/hooks/useEmployees.tsx` | **การทำงาน:** สร้าง Record ในตาราง `employees` และ `employees_profile` พร้อมกัน<br>**วัตถุประสงค์:** เพื่อเพิ่มพนักงานเข้าสู่ระบบโดยระบุอีเมลและทักษะ (Skill) เบื้องต้นไว้ล่วงหน้า |
| `updateEmployee` | `src/hooks/useEmployees.tsx` | **การทำงาน:** อัปเดตข้อมูลพนักงาน ทั้งบทบาท (Role) และข้อมูลส่วนตัว<br>**วัตถุประสงค์:** เพื่อปรับปรุงข้อมูลพนักงานให้เป็นปัจจุบันเมื่อมีการย้ายแผนกหรือเปลี่ยนหน้าที่ |
| `deleteEmployee` | `src/hooks/useEmployees.tsx` | **การทำงาน:** ลบข้อมูลพนักงานและโปรไฟล์ออกจากระบบถาวร<br>**วัตถุประสงค์:** เพื่อทำความสะอาดข้อมูลเมื่อพนักงานลาออกหรือระบุข้อมูลผิดพลาด |

---

## 🔐 2. การควบคุมการเข้าถึง (Access Control)
ระบบระงับและกู้คืนสถานะการเข้าใช้งาน

| ชื่อฟังก์ชัน / Mutation | Path ไฟล์ | รายละเอียดการทำงานและวัตถุประสงค์ |
| :--- | :--- | :--- |
| `suspendEmployee` | `src/hooks/useEmployees.tsx` | **การทำงาน:** เปลี่ยนสถานะพนักงานเป็น `suspended` (ระงับการใช้งาน)<br>**วัตถุประสงค์:** เพื่อตัดการเชื่อมต่อพนักงานจากระบบชั่วคราวโดยไม่ต้องลบข้อมูลทิ้ง |
| `reactivateEmployee` | `src/hooks/useEmployees.tsx` | **การทำงาน:** เปลี่ยนสถานะพนักงานกลับเป็น `active`<br>**วัตถุประสงค์:** เพื่อคืนสิทธิ์การเข้าใช้งานให้กับพนักงานที่เคยถูกระงับ |

---

## ✅ 3. ระบบอนุมัติพนักงานใหม่ (Approval Workflow)
ขั้นตอนคัดกรองพนักงานใหม่ที่สมัครเข้ามาในระบบ

| ชื่อฟังก์ชัน / Mutation | Path ไฟล์ | รายละเอียดการทำงานและวัตถุประสงค์ |
| :--- | :--- | :--- |
| `approveEmployee` | `src/hooks/useEmployees.tsx` | **การทำงาน:** เปลี่ยนสถานะการอนุมัติเป็น `approved`<br>**วัตถุประสงค์:** เพื่ออนุญาตให้พนักงานที่สมัครเข้ามาใหม่สามารถเริ่มใช้งานระบบได้เป็นครั้งแรก |
| `rejectEmployee` | `src/hooks/useEmployees.tsx` | **การทำงาน:** เปลี่ยนสถานะการอนุมัติเป็น `rejected`<br>**วัตถุประสงค์:** เพื่อปฏิเสธการสมัครของบุคคลที่ไม่มีสิทธิ์เข้าถึงระบบ |

---

## 🛡️ 4. ระบบบันทึกความปลอดภัย (Security Audit)
การบันทึกเหตุการณ์สำคัญที่เกิดขึ้นในหน้าจัดการพนักงาน

| หัวข้อ Logic | รายละเอียดการทำงาน | วัตถุประสงค์ |
| :--- | :--- | :--- |
| **Audit Logging** | มีการเรียกใช้ฟังก์ชัน `auditSecurity` ทุกครั้งที่มีการ Approve, Reject, หรือ Suspend พนักงาน | เพื่อสร้างบันทึก (Audit Trail) ว่าใครเป็นผู้อนุมัติหรือระงับใคร และทำเมื่อไหร่ เพื่อความโปร่งใสของระบบ |

---

> [!TIP]
> **Data Integrity:** ในฟังก์ชัน `createEmployee` มีระบบ **Safe Rollback** หากสร้างข้อมูลพนักงานสำเร็จแต่สร้างโปรไฟล์ไม่สำเร็จ ระบบจะทำการลบพนักงานคนนั้นทิ้งทันที เพื่อป้องกันข้อมูลขยะ (Orphan Data) ในฐานข้อมูล
